# Contribute to PortMaster

PortMaster is built by volunteers who love handheld gaming and open-source software. There are many ways you can help make PortMaster better!

---

## Ways to Contribute

### Port Games

Help bring more games to handheld devices by creating new ports.

[:material-gamepad-variant: Start Porting](porting/what-can-be-ported.md){ .md-button .md-button--primary }

**What you'll do:**

- Port games using engines like Love2D, GameMaker, and more
- Package ports for distribution
- Test on different devices

**Resources:**

- [What Can Be Ported?](porting/what-can-be-ported.md)
- [Engine Guides](porting/engines/love2d.md) (Love2D, GameMaker Studio)
- [Build Environments](porting/build-environments.md)
- [Packaging Guide](porting/packaging.md)

---

### Improve the GUI

Help make PortMaster's interface better and more accessible.

[:material-palette: Customize Themes](gui/theme-engine.md){ .md-button }
[:material-translate: Add Translations](gui/translation.md){ .md-button }

**What you'll do:**

- Create new themes for the PortMaster app
- Translate the interface to other languages
- Improve user experience

---

### Write Documentation

Help others by improving our guides and documentation.

[:material-file-document-edit: Edit Docs](https://github.com/PortsMaster/PortMaster-New){ .md-button }

**What you'll do:**

- Fix errors and typos
- Add missing information
- Create new guides
- Improve clarity

---

### Join the Community

Connect with other contributors and help fellow users.

[:fontawesome-brands-discord: Join Discord](https://discord.gg/eqjK6yNQS4){ .md-button }
[:fontawesome-brands-github: GitHub Discussions](https://github.com/PortsMaster/PortMaster-GUI/discussions){ .md-button }

**How to help:**

- Answer questions from users
- Test new ports and features
- Share feedback and ideas
- Report bugs

---

## Getting Started

**New to contributing?** Here's what to do:

1. Join our [Discord](https://discord.gg/eqjK6yNQS4) and introduce yourself
2. Pick an area that interests you above
3. Read the relevant guides
4. Start small - fix a typo, test a port, or ask questions
5. Work your way up to bigger contributions
